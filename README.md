# BrightLearn- Data Agent Project

## Project Objective

The purpose of this project was to design and build a working data agent using Databricks while applying the skills learned at BrightLearn. The project focused on creating a Sales Data Agent capable of analysing retail sales data and answering important business questions related to sales performance, customer behaviour, and product trends.

The agent was designed to assist the store owners, managers, and decision makers by providing insights that can support better business planning and smarter decision making.

## Tools Used

The following tools and technologies were used during the project:

* Databricks for the workspace environment, SQL Editor, and AI agent configuration
* Databricks visualisation tools for creating charts and graphs
* BrightLearn Student portal for the retail sales dataset


## Dataset Overview

The dataset used in this project was provided by BrightLearn and contains 1,000 retail transaction records collected between January 2023 and January 2024.

The dataset includes the following fields:

| Column           | Type    |
| ---------------- | ------- |
| Transaction ID   | Integer |
| Date             | Date    |
| Customer ID      | String  |
| Gender           | String  |
| Age              | Integer |
| Product Category | String  |
| Quantity         | Integer |
| Price per Unit   | Integer |
| Total Amount     | Integer |

The dataset contains three product categories: Electronics, Clothing, and Beauty. It also includes Male and Female customer records. Each transaction belongs to a unique customer, giving a total of 1,000 customers across 1,000 transactions.


## Steps Followed

The project was completed through the following steps:

1. Uploaded the retail sales dataset into Databricks and registered it as a table called `workspace.default.retail_sales_dataset`

2. Explored the dataset using SQL queries to understand the structure, data types, column names, and date range

3. Validated the dataset by checking row counts, customer records, product categories, and gender values

4. Configured the Sales Intelligence Agent inside the Databricks AI Agent interface by defining the role, responsibilities, rules, and limitations of the agent

5. Added 12 (only 10 was required) business related questions as starter prompts for the agent

6. Tested the agent by asking 10 questions and reviewing the responses generated

7. Wrote SQL queries to validate every response provided by the agent

8. Created visualisations from the SQL results and compared them with the outputs generated by the agent

9. Analysed the results and documented key findings together with business recommendations


## Agent Instructions

### Agent Name:

Sales Intelligence Agent

### Role:

The agent acts as a retail sales analyst responsible for answering questions related to sales performance, customer purchasing behaviour, pricing trends, and product category performance using the `retail_sales_dataset`.

### Intended Users:

The agent was designed for store owners, managers, and board members who need support in identifying business trends, peak sales seasons, and product performance in order to make informed business decisions.

### Query Handling

If a question cannot be answered because of missing or invalid data, the agent only returns results based on valid records. It also provides a short explanation to guide the user on possible next steps.

### Limitations

The dataset only contains sales information. The agent cannot answer questions related to stock availability, store operations, supplier information, or promotional discounts.


## Sample Questions Tested

The following questions were used to test the agent:

1. When is the best time to run store promotions?
2. What is the best selling product?
3. Which month has the most sales?
4. Which gender purchases the most in the store?
5. Which age group purchases the most?
6. Which product sells the least?
7. When do peak sales occur during the month?
8. What is the monthly average sales value of the store?
9. What is the annual total revenue of the store?
10. How many customers visit the store daily?
11. How many customers spent more than $500 in a single transaction?
12. Which age group spends the most per transaction?

The first 3 answers generated by the agent were validated using  SQL queries and visual comparisons through charts and graphs.


## Key Insights

### 1. Product performance was fairly balanced

Electronics generated the highest revenue at $156,905, followed closely by Clothing at $155,580. Beauty produced $143,515 in revenue. Even though Electronics performed the best overall, the difference between the highest and lowest category was relatively small over the full year.

One interesting observation was that Beauty had fewer transactions than the other categories, even though customers tended to spend more per visit there compared to Clothing. This suggests that the challenge may not be customer spending, but rather attracting more customers to the category.

### 2. Sales performance changed significantly across different months

May recorded the strongest performance with revenue of $53,150, while September was the weakest month with only $23,620 in sales. The difference between these months was substantial and highlights the impact seasonal trends can have on business performance.

This type of variation can affect staffing, stock planning, and promotional strategies throughout the year.

### 3. Younger customers showed strong spending potential

Customers between the ages of 45 and 54 contributed the highest number of transactions and overall revenue, making them the most consistent customer group.

However, customers under the age of 25 had the highest average spend per transaction at $501, despite visiting less frequently. This suggests there is an opportunity to increase revenue by finding ways to attract younger shoppers more often.

### 4. Saturdays performed best while Thursdays were the weakest

Saturday generated the highest revenue at $78,815, making it the strongest trading day of the week. Thursday recorded the lowest revenue at $53,008.

This pattern suggests that the business could benefit from running promotions or special offers during quieter mid week periods to encourage more customer activity.

### 5. Customer activity increased near month end

Transaction activity increased noticeably around the 23rd and 24th of each month. This pattern suggests that many customers shop shortly after receiving their salaries.

Understanding this trend can help the store prepare better through improved stock planning and staffing during busier periods.


## Business Recommendations

Based on the findings from the analysis, the following recommendations were made:

* Introduce promotional campaigns during weaker months such as September, March, and April to improve sales performance

* Create special Thursday promotions or loyalty rewards to increase mid week customer traffic

* Focus more marketing efforts on customers under 25 through social media campaigns and student focused promotions

* Improve visibility of Beauty products through bundle offers and category specific marketing campaigns

* Prepare for month end shopping increases by ensuring shelves are fully stocked and staffing levels are sufficient

* Maintain strong service and stock availability on Saturdays since it is the highest performing day of the week


## Conclusion

This project was a valuable learning experience that combined SQL querying, AI agent configuration, data validation, and data visualisation into one complete solution.

One of the most important lessons learned during the project was the importance of validating AI generated answers with SQL queries. Comparing the agent responses with actual query results helped build confidence in the accuracy of the analysis and reinforced the importance of verification when working with AI systems.

Data visualisation was also a new skill introduced during the project. Choosing the correct chart type for each business question was challenging at first, but it added meaningful value to the final analysis and made the results easier to understand.

Looking back, some of the questions could have been written more specifically from the beginning. A few broad questions produced highly detailed responses that required additional interpretation. More focused questions would make validation and analysis even more efficient.

Overall, this project demonstrated how a well designed data agent can support real business decision making by transforming raw sales data into useful and actionable insights.
